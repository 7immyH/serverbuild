Assembly order

Follow this sequence exactly. Work on a clean, static-free surface. Touch bare metal (the PSU case) before handling any component to discharge static. Assemble the CPU, cooler and RAM on the board outside the case first.

1

Prep the case

Remove both side panels. Verify ATX standoffs are pre-installed for ATX layout. Remove I/O shield knock-outs for your ports. Lay the case on its side so the motherboard tray faces up.

Set the MSI B550-A PRO box flat on your desk as a work surface — assembling CPU and RAM outside the case is much easier than working inside it.
2

Install CPU into motherboard

Lift the AM4 ZIF lever fully. Align the gold triangle marker on the Ryzen 5600 with the triangle on the socket corner. Lower the CPU — it drops in with zero force, guided by two notches. Close the lever firmly.

Never touch the gold pins on the underside of the CPU. Hold the chip by its edges only. One bent pin can kill the board and the CPU both.
3

Install CPU cooler (Wraith Stealth)

The Wraith Stealth has pre-applied thermal paste — do not add more. Align the four push-pins over the AM4 backplate holes. Press down and twist each clockwise until it clicks. Plug the 4-pin fan cable into CPU_FAN1 — the header nearest the top-center of the board.

Tighten in an X-pattern: top-left → bottom-right → top-right → bottom-left. This spreads pressure evenly across the CPU die and prevents warping the board.
4

Install RAM in dual-channel slots

On the MSI B550-A PRO use slots A2 and B2 — the second and fourth slots from the left edge (check silkscreen labeling on the board). This enables dual-channel mode. Press each stick firmly and evenly until both side clips snap in fully.

RAM requires more force than expected — around 20–25 lbs per stick. If the side clip doesn't snap, the stick is not fully seated and the system will not POST. Do not skip past this.
5

Install NVMe SSD

Remove the M.2_1 heatsink (closest to the CPU — one screw). Remove the retaining screw at the far end of the slot. Slide the WD SN580 in at a 30° angle, press down flat, and replace the retaining screw. Reinstall the heatsink over the drive on its thermal pad.

Always reinstall the M.2 heatsink. The B550-A PRO's heatsink keeps the NVMe significantly cooler during sustained read/write under Proxmox and video editing workloads.
6

Install motherboard into case

Press the I/O shield from the motherboard box into the rear of the case — it snaps in with firm pressure. Lower the board into the case aligning the rear ports through the shield. Match screw holes to standoffs. Install all 9 ATX screws with a magnetic screwdriver — snug, not over-torqued. Work center-out.

7

Install IronWolf drives

Slide the two IronWolf 8TB drives into 3.5" bays 1 and 2. Secure with four screws per drive through the case side rails. Connect SATA data cables to SATA_1 and SATA_2 on the board (the two ports furthest from the GPU slot). Connect SATA power from the PSU.

8

Install PSU

Mount the Corsair RM750x in the PSU shroud at the bottom of the case, fan facing down toward the floor vent. Secure with 4 screws through the rear panel. Attach only the cables you need before the GPU goes in — 24-pin ATX, 8-pin CPU EPS, and two PCIe 8-pin cables for the GPU.

Route cables through the back panel grommets before mounting the GPU. Cable management is much harder with the GPU in place.
9

Install RTX 4070 Ti Super

Remove the two PCIe slot cover plates from the case rear panel aligned with the top x16 slot. Press the PCIe slot retention clip outward. Align the GPU and press down firmly and evenly until the clip clicks. Screw the bracket into the case. Connect two PCIe 8-pin power cables from the PSU.

The RTX 4070 Ti Super requires two 8-pin PCIe power connectors. Never substitute the CPU EPS cable — they look nearly identical but carry different voltages. PCIe cables on the RM750x are clearly labeled.
10

Connect Intel i225-V 2.5GbE NIC

Install the 2.5GbE card in any available PCIe x1 or x4 slot below the GPU. Remove that slot cover, press the card in until the clip clicks, screw the bracket in. Connect your Ethernet cable to this card — not the motherboard's 1GbE port.

11

Connect front panel headers and close up

The JFP1 header is in the bottom-right of the MSI B550-A PRO. Connect: power switch (no polarity), reset switch (no polarity), power LED (polarity marked), HDD LED. Connect USB 3.0 case header to USB3_1, HD Audio to JAUD1. Do a final cable inspection — no wires near fans. Connect monitor to the RTX 4070 Ti Super's DisplayPort output.

The MSI EZ Debug LEDs (CPU, DRAM, VGA, BOOT near the top-right of the board) show exactly which component is causing a problem if the system doesn't POST on first boot.

BIOS settings

Press Delete repeatedly on boot to enter BIOS. Press F7 to switch to Advanced Mode. Make all changes below in order, then press F10 to save and restart.

OC → Memory → XMP

Enable A-XMP Profile

Set to Profile 1 — unlocks DDR4 to its rated 3200 MHz speed. Without this it defaults to 2133 MHz, a significant performance penalty for everything running on this machine.

Settings → Advanced → PCI Subsystem

Above 4G Decoding

Set to Enabled — required for the RTX 4070 Ti Super to access all 16GB of VRAM properly. Also needed for Proxmox GPU passthrough to work correctly.

Settings → Advanced → PCI Subsystem

Re-Size BAR Support

Set to Enabled — allows the CPU to access the full GPU frame buffer at once. Measurable performance boost in rendering, viewport work and video encoding.

Settings → Advanced → AMD CBS → CPU Common Options

Global C-State Control

Set to Enabled — allows the CPU to idle down efficiently when not under load. Important for a 24/7 machine — reduces heat and electricity consumption at idle.

Settings → System → Windows OS Configuration

BIOS UEFI / CSM Support

Set to UEFI — required for NVMe boot drives to operate at full speed and for Proxmox UEFI VMs (needed for Windows 11 and GPU passthrough).

Settings → Advanced → ACPI Settings

BIOS ACPI Sleep State

Set to S3 (Suspend to RAM) if you want the machine to sleep when not in use. Set to Disabled if running as a 24/7 server — prevents the host from sleeping while VMs are running.

Boot

Boot Device Priority

When installing Proxmox: set USB as first. After Proxmox install: set WD SN580 NVMe as first and remove USB from the list. Do this before the IronWolf drives show up as boot candidates.

After all changes press F10 → Save and Exit. The system restarts. If the display comes on and fans spin normally, hardware is good. Proceed to flash the Proxmox USB installer.
If no display after POST — check EZ Debug LEDs on the board. DRAM lit = reseat RAM sticks. VGA lit = reseat GPU or check PCIe power cables. CPU lit = recheck 8-pin EPS cable and cooler mounting.

SolidWorks GPU configuration

If you use SolidWorks inside the Windows VM, configure these settings after Windows and NVIDIA drivers are installed. The RTX 4070 Ti Super is not on NVIDIA's certified SolidWorks list but performs excellently with this setup.

1

Install NVIDIA Studio Driver

Download from nvidia.com → GeForce → RTX 4070 Ti Super → your Windows version. Select Studio Driver (not Game Ready). During installation choose Custom Install → Clean Install.

Studio drivers are optimized for workstation applications including SolidWorks, DaVinci Resolve and Premiere Pro. They have significantly fewer rendering artifacts in CAD viewports than Game Ready drivers.
2

Force SolidWorks to use the RTX 4070 Ti Super

NVIDIA Control Panel → Manage 3D Settings → Program Settings → Add → browse to sldworks.exe at C:\Program Files\SOLIDWORKS Corp\SOLIDWORKS\. Set Preferred graphics processor to your RTX 4070 Ti Super. Save.

3

Enable hardware acceleration in SolidWorks

Tools → Options → System Options → Performance: confirm "Use software OpenGL" is unchecked. Tools → Options → System Options → Display: set image quality to balanced (not maximum — maximum tanks frame rate on complex assemblies).

4

Suppress the unsupported GPU warning

Create a system environment variable: Windows Search → Environment Variables → System Variables → New.

Variable name: SOLIDWORKS_USE_GPU_RENDERING Variable value: 1

Restart SolidWorks after saving. This removes the startup warning and enables smoother viewport rendering.

5

Recommended performance settings

Tools → Options → Performance: ✓ Automatically load components lightweight: ON ✓ Use shaded mode for assemblies over 250 components ✓ Dynamic highlight from graphics view: ON ✗ Verify Z-buffer depth on every open: OFF ✓ Level of detail: Enabled

Proxmox VE — installation

Proxmox runs directly on the bare metal NVMe SSD. All other systems — Windows, Home Assistant, Docker — run as VMs or containers inside it. One web UI manages everything.

1

Download and flash Proxmox VE

Download the latest Proxmox VE ISO from proxmox.com/downloads. Flash to a USB drive using Rufus (Windows) — select DD mode, not ISO mode. Minimum 8GB USB.

As of 2026, Proxmox VE 8.x is current. Get the VE edition — not Proxmox Backup Server.
2

Boot and run the installer

Press Delete on boot → set USB as first boot device → save and restart. Select "Install Proxmox VE (Graphical)" from the boot menu.

At the disk selection screen choose your WD SN580 NVMe — not the IronWolf drives. Confirm the size shown matches your 1TB NVMe before proceeding.
3

Filesystem and disk settings

At the disk setup screen click Options. Set filesystem to ext4 — simple and reliable for the OS disk. Leave hdsize at default. The installer creates boot, swap and root partitions automatically.

Save ZFS for the IronWolf NAS pool, not the boot disk. ZFS on the boot disk adds complexity and is not needed here.
4

Network, hostname and passwords

Set a static management IP: 192.168.1.10. Set hostname: pve.local. Set a strong root password and an email address for alerts. Confirm your 2.5GbE NIC is selected as the management interface.

5

First login and community repo setup

After install, remove USB and reboot. From any browser go to https://192.168.1.10:8006. Log in as root. Dismiss the "No valid subscription" popup. Then in Node → Shell:

sed -i 's/^deb/#deb/' /etc/apt/sources.list.d/pve-enterprise.list echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" \ > /etc/apt/sources.list.d/pve-community.list apt update && apt dist-upgrade -y reboot
6

Enable IOMMU for GPU passthrough

Do this before creating any VMs. In the Proxmox shell:

nano /etc/default/grub # Change GRUB_CMDLINE_LINUX_DEFAULT to: GRUB_CMDLINE_LINUX_DEFAULT="quiet amd_iommu=on iommu=pt" update-grub echo "vfio" >> /etc/modules echo "vfio_iommu_type1" >> /etc/modules echo "vfio_pci" >> /etc/modules update-initramfs -u -k all reboot
7

Create the ZFS mirror pool (IronWolf drives)

Proxmox web UI → Node → Disks → ZFS → Create ZFS. Select both IronWolf drives. RAID Level: Mirror. Compression: lz4. Name: datapool. Or via the full CLI setup covered in the ZFS section.

8

Verify IOMMU groups for GPU

find /sys/kernel/iommu_groups/ -type l | sort -V | head -40 # Your RTX 4070 Ti Super should appear in its own group # If it shares a group with other devices, passthrough may need ACS override lspci -nn | grep -i nvidia # Note both hex IDs: VGA [10de:XXXX] and Audio [10de:XXXX]

GPU passthrough — RTX 4070 Ti Super

GPU passthrough gives the Windows VM direct native access to the RTX 4070 Ti Super — identical performance to bare metal. Required: IOMMU enabled in Proxmox install section first.

1

Blacklist the GPU from the Proxmox host

# Find GPU PCI IDs lspci -nn | grep -i nvidia # e.g. 09:00.0 VGA [10de:2782] 09:00.1 Audio [10de:22bc] echo "options vfio-pci ids=10de:2782,10de:22bc" > /etc/modprobe.d/vfio.conf echo "blacklist nouveau" >> /etc/modprobe.d/blacklist.conf echo "blacklist nvidia" >> /etc/modprobe.d/blacklist.conf update-initramfs -u -k all reboot
Replace 10de:2782 and 10de:22bc with your actual GPU PCI IDs from lspci output above.
2

Create Windows 11 VM with correct settings

Machine: q35 BIOS: OVMF (UEFI) CPU Type: host CPU Cores: 6 RAM: 20480 MB Disk: 200 GB, VirtIO SCSI, on local-lvm Network: VirtIO, bridge vmbr0 Display: Default (VGA) during install — change later
3

Add GPU PCI device to the VM

VM Hardware tab → Add → PCI Device → select the RTX 4070 Ti Super VGA entry. Enable:

✓ All Functions (passes GPU + HDMI audio together) ✓ ROM-Bar ✓ PCI-Express ✗ Primary GPU (enable this AFTER Windows is fully set up)
4

Install Windows and NVIDIA drivers

Boot VM from a Windows 11 ISO (upload to Proxmox local storage first). Attach the VirtIO driver ISO too — Windows won't see the VirtIO disk without it. Install Windows normally. Once on the desktop: install VirtIO guest tools, then NVIDIA Studio Driver. The GPU should appear in Device Manager.

Connect your monitor to the RTX 4070 Ti Super DisplayPort before starting the VM. Once NVIDIA drivers install, output switches from the virtual Proxmox display to the physical GPU.
5

Hide hypervisor from NVIDIA driver (if needed)

If you see "NVIDIA Installer cannot continue" or Error 43 in Device Manager:

nano /etc/pve/qemu-server/100.conf # Add this line: args: -cpu 'host,+kvm_pv_unhalt,+kvm_pv_eoi,hv_vendor_id=NV43FIX,kvm=off'
With recent NVIDIA drivers (550+) and Proxmox 8.x this is often no longer needed. Try without it first — only add if you hit the error.
6

Set as primary GPU and finalize

Verify in DaVinci Resolve: Preferences → Memory and GPU → GPU Processing Mode shows CUDA with RTX 4070 Ti Super. Then in Proxmox → VM Hardware → edit the PCI device → enable Primary GPU. Set VM Display to None. From now on the VM boots directly to the physical GPU output.

ZFS pool — full setup

ZFS runs on the Proxmox host and manages your two IronWolf drives. Covers pool creation, datasets, scrubs, snapshots and failure alerts.

1

Identify drives by stable by-id paths

lsblk -o NAME,SIZE,MODEL,SERIAL ls -la /dev/disk/by-id/ | grep -i seagate # Use the full ata-ST8000VN004-xxxx_SERIALNO paths in the pool create
Always use by-id paths — not /dev/sda or /dev/sdb. Device names can change between reboots; by-id paths are permanently tied to the drive's serial number.
2

Create the ZFS mirror pool

zpool create -f \ -o ashift=12 \ -O compression=lz4 \ -O atime=off \ -O xattr=sa \ -O dnodesize=auto \ datapool mirror \ /dev/disk/by-id/ata-ST8000VN004-xxxx_SERIAL1 \ /dev/disk/by-id/ata-ST8000VN004-xxxx_SERIAL2 zpool status datapool # should show: state: ONLINE
3

Create datasets

zfs create datapool/media zfs create datapool/projects zfs create datapool/backups zfs create datapool/documents zfs create datapool/docker zfs set compression=off datapool/media # already compressed content zfs set quota=2T datapool/backups # prevent backups filling the pool
4

Add datapool to Proxmox storage

pvesm add zfspool datapool --pool datapool --sparse 1 --content images,rootdir
5

Configure monthly scrubs

# Verify scrub timer is active systemctl status zfs-scrub.timer # Run a manual scrub now to confirm pool health zpool scrub datapool && zpool status datapool
6

Automatic snapshots with Sanoid

apt install sanoid -y
/etc/sanoid/sanoid.conf
[datapool/projects] use_template = production [datapool/documents] use_template = production [template_production] hourly = 6 daily = 14 weekly = 4 monthly = 3 autosnap = yes autoprune = yes
systemctl enable sanoid.timer --now
Recover deleted files from: /datapool/projects/.zfs/snapshot/ — just copy the file back out. No commands needed.
7

ZFS failure email alerts

/etc/zfs/zed.d/zed.rc — edit these values
ZED_EMAIL_ADDR="you@youremail.com" ZED_EMAIL_VERBOSE=1 ZED_NOTIFY_INTERVAL_SECS=3600
systemctl enable zed --now

Backblaze B2 — cloud backup

Backblaze B2 is ~$6/TB/month with no egress fees to Cloudflare. The cheapest reliable offsite storage available. Protects against fire, theft and catastrophic hardware failure.

Cost estimate and full 3-2-1 summary

# Estimated monthly B2 cost Documents (50 GB) ~$0.30/month HA config (1 GB) ~$0.01/month Projects (200 GB) ~$1.20/month Total: ~$1.50 – $5/month # Full 3-2-1 strategy Copy 1: ZFS RAID 1 on IronWolf drives — primary, redundant, instant Copy 2: Duplicati → USB external drive — nightly, 30-day retention Copy 3: Duplicati → Backblaze B2 — nightly, 90-day, offsite, encrypted Alerts: HA webhook on any failure → push notification Testing: Full restore test every 3 months
Don't back up your full media library to B2 — expensive and slow to restore. Back up only irreplaceable data: documents, projects, photos, HA config. Media can be re-downloaded or re-ripped.

Ongoing maintenance

Regular tasks that keep the stack healthy, secure and backed up. A well-maintained Proxmox server runs for years without issues.

Weekly (automated)
1

Proxmox VM backup schedule

Datacenter → Backup → Add. Configure to back up all VMs to datapool/backups every Sunday at 02:00. Retention: keep 3 copies. This is separate from ZFS snapshots — a full VM restore point.

vzdump 100 101 200 --storage datapool --compress zstd --mode snapshot
Monthly
2

System updates

# Proxmox host apt update && apt dist-upgrade -y # Inside LXC container apt update && apt upgrade -y # Docker containers (manual pull to review changes) cd /opt/docker/nginx-proxy && docker compose pull && docker compose up -d cd /opt/docker/jellyfin && docker compose pull && docker compose up -d cd /opt/docker/duplicati && docker compose pull && docker compose up -d docker system prune -f
3

ZFS health check

zpool status -v datapool # full health — look for ONLINE zfs list -r -o name,used,avail,refer datapool # dataset usage zpool scrub datapool # kick off a manual scrub
If you ever see "DEGRADED" — a drive has failed. Replace it immediately. ZFS RAID 1 protects your data during the failure but you have zero redundancy until the new drive rebuilds.
4

Replace a failed drive

# Identify failed drive zpool status datapool # Power down → replace drive → power on → resilver zpool replace datapool \ /dev/disk/by-id/OLD_DRIVE_ID \ /dev/disk/by-id/NEW_DRIVE_ID # Monitor resilver — typically 4–8 hrs for 8TB zpool status datapool
5

Restore test reminder

# Every 3 months — test restore from Duplicati # Duplicati UI → Restore → select a file → restore to /tmp/restore-test # Verify file integrity → delete /tmp/restore-test # Also test: restore one file from a ZFS snapshot ls /datapool/projects/.zfs/snapshot/ cp /datapool/projects/.zfs/snapshot/SNAPSHOT_NAME/filename.ext /tmp/
