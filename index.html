Assembly order

Follow this sequence to avoid damage and make installation easier. Work on a static-free surface. Touch bare metal (the PSU case) before handling any component. Assemble CPU and RAM on the motherboard outside the case first.

1

Prep the case

Remove both side panels from the Fractal Define 7. Verify ATX standoffs are installed for ATX layout. Remove the I/O shield knockout. Set the case on its side so the motherboard tray faces up. Route cables now before components are in.

The Define 7's modular drive cage system is pre-configured for ATX. If you want maximum drive bay access later, leave the bottom 3.5" cage in place and populate from the back.
2

Install CPU into motherboard (outside case)

Lay the MSI B650 Gaming Plus WiFi on its box on a flat surface. Lift the AM5 ZIF lever, align the Ryzen 9 7900 triangle marker to the socket triangle, lower into socket, and close the lever. No force needed — if it doesn't drop in, recheck alignment.

AM5 uses a contact-frame design — the CPU pins are on the motherboard socket, not the chip. Do not touch the socket contacts.
3

Install CPU cooler (Wraith Prism)

Apply the included thermal paste (or a pea-sized dot of your own). The Wraith Prism ships with AM5 brackets. Align the four mounting screws to the AM5 backplate holes and tighten in a cross pattern. Plug the fan header into CPU_FAN1.

The AM5 mounting holes are the same as AM4. The Wraith Prism included with the Ryzen 9 7900 (non-X) fits directly — no adapter needed.
4

Install RAM — DIMM slots A2 and B2

Consult the B650 manual for dual-channel layout — typically slots 2 and 4 (labeled A2/B2). Insert the G.Skill Flare X5 2×32GB sticks into those slots. Press firmly until both side clips lock. DDR5 is notched so it can only go in one direction.

5

Install NVMe drives (M.2 slots)

Slot 1 (top M.2, near CPU): WD SN580 1TB — this is the boot/Proxmox drive. Slot 2 (second M.2): WD SN580 500GB — dedicated to Windows VM. Remove the M.2 heatsink, slide the drive in at 30°, lower flat, secure with screw, replace heatsink.

6

Seat motherboard in case

Snap the B650's I/O shield into the case opening. Lower the board onto the standoffs, aligning all screw holes. Use 9 screws (ATX layout). Don't overtighten — snug is enough for good contact.

7

Install Intel i225-V NIC

Seat into the first available PCIe ×1 slot (avoid blocking the main ×16 GPU slot for Phase 2). Secure the bracket screw. Remove the corresponding I/O slot cover from the Define 7's rear.

8

Install IronWolf drives (3.5" bays)

Use the Define 7's tool-free drive trays. Slide each IronWolf 8TB into a tray, secure with the rubber-dampened screws on the tray sides, and slide the tray into the cage until it clicks. Connect SATA III cables (right-angle on drive end) to motherboard SATA ports 1 and 2.

9

Install PSU (Corsair RM750x)

Mount fan-down in the Define 7's bottom PSU shroud (fan exhausts out the back). Connect cables: 24-pin ATX, 8-pin CPU (EPS), SATA power to both IronWolf drives. Leave GPU power cables aside — not needed until Phase 2. Route all cables through grommets before closing panels.

✓

Final cable check & panel close

Verify: 24-pin ATX, CPU 8-pin, both SATA power, both SATA data, front panel headers (PWR, RST, HDD LED, USB, audio), CPU FAN header. Tuck cables behind the tray. Reinstall both side panels. Connect HDMI from motherboard HDMI port to monitor.

BIOS settings

MSI B650 Gaming Plus WiFi — apply these settings before Proxmox installation. Enter BIOS by pressing Del or F2 during POST. Press F7 for Advanced Mode.

CPU & Memory

OC › Memory → XMP/EXPO

Enable XMP / EXPO Profile

Set to EXPO Profile 1. This enables the G.Skill Flare X5 at DDR5-6000 CL36 as rated. Without this, RAM runs at DDR5-4800 (JEDEC default).

Settings › Advanced › CPU Configuration

SVM Mode (Virtualization)

Set to Enabled. Required for Proxmox KVM hypervisor.

Settings › Advanced › Integrated Graphics Configuration

Initiate Graphic Adapter

Set to iGPU. This ensures Proxmox outputs display through the motherboard HDMI permanently, leaving the PCIe slot entirely for the Phase 2 GPU passthrough.

PCIe & IOMMU

Settings › Advanced › PCI Subsystem Settings

Above 4G Memory / Crypto Currency Mining

Set to Enabled. Required for GPU passthrough in Phase 2 (Above 4G Decoding).

Settings › Advanced › PCI Subsystem Settings

Re-Size BAR Support

Set to Enabled. Enables Resizable BAR (ReBAR) — the RTX 4070 Ti Super supports this and gains performance benefit.

Settings › Advanced › AMD CBS › NBIO Common Options

IOMMU

Set to Enabled. Required for VFIO GPU passthrough. Must be set in BIOS before enabling in GRUB.

Power & Storage

Settings › Advanced › Power Management

Restore on AC Power Loss

Set to Power On. Proxmox server should auto-restart after power outage (even with UPS, belt-and-suspenders).

Settings › Advanced › SATA Configuration

SATA Mode

Verify AHCI. NVMe drives are unaffected; IronWolf SATA drives need AHCI for Proxmox detection.

After saving BIOS settings (F10), boot from the Proxmox USB installer. The two NVMe drives and two IronWolf drives should all appear in the installer.

IOMMU & VFIO prep

Configure IOMMU and VFIO modules in Phase 1 — before the GPU arrives. There's no harm in doing this without a GPU installed, and it saves a reboot cycle when Phase 2 hardware arrives.

1

Enable IOMMU in GRUB

Edit the GRUB configuration to pass amd_iommu=on and iommu=pt to the kernel on every boot.

/etc/default/grub

# Find this line and update it: GRUB_CMDLINE_LINUX_DEFAULT="quiet amd_iommu=on iommu=pt" # Then update grub and reboot: update-grub reboot

2

Load VFIO kernel modules

Add all four VFIO modules to /etc/modules so they load at every boot. This prepares the system to claim the GPU from the host when Phase 2 hardware is installed.

/etc/modules — append these lines

vfio vfio_iommu_type1 vfio_pci vfio_virqfd

echo "vfio vfio_iommu_type1 vfio_pci vfio_virqfd" >> /etc/modules update-initramfs -u -k all
3

Verify IOMMU groups (after reboot)

Run this script to confirm IOMMU is active and groups are isolated. Each GPU function should appear in its own group.

#!/bin/bash # Check IOMMU groups — run after reboot for d in /sys/kernel/iommu_groups/*/devices/*; do n=${d#*/iommu_groups/*}; n=${n%%/*} printf 'IOMMU Group %s ' "$n" lspci -nns "${d##*/}" done
Look for the RTX 4070 Ti Super entries (when installed in Phase 2) each in their own group. If the GPU shares a group with other critical devices, you may need ACS override — but B650 + Ryzen 9 7900 typically produces clean isolation.
4

Blacklist NVIDIA GPU from Proxmox host (Phase 2)

After installing the RTX 4070 Ti Super, prevent Proxmox from loading NVIDIA drivers so VFIO can claim the card exclusively.

# Get GPU PCI IDs lspci -nn | grep -i nvidia # Note the IDs in format [10de:xxxx] — add both GPU and audio function # Create VFIO config file cat > /etc/modprobe.d/vfio.conf << 'EOF' options vfio-pci ids=10de:2782,10de:22bc # Replace 2782,22bc with your actual GPU and audio IDs EOF # Blacklist nouveau and nvidia from host cat > /etc/modprobe.d/blacklist-nvidia.conf << 'EOF' blacklist nouveau blacklist nvidia blacklist nvidiafb EOF update-initramfs -u -k all reboot

Proxmox VE install

Proxmox VE 8.x on the WD SN580 1TB NVMe. The 500GB NVMe will be used as the Windows VM disk and is left unformatted by the installer.

1

Run the Proxmox installer

Boot from the Proxmox USB. Choose Graphical Install. Target disk: the 1TB NVMe. Set ZFS RAID0 on the single 1TB drive (not a ZFS mirror — NAS redundancy is handled separately by the IronWolf pool). Set hostname, IP, and gateway.

Recommended static IP for Proxmox host: 192.168.1.10 — matches the network plan in this guide.
2

Post-install: disable enterprise repo, enable community repo

The enterprise repo requires a paid subscription. Switch to the community (no-subscription) repo to get updates.

# Disable enterprise repo sed -i 's/^deb/#deb/' /etc/apt/sources.list.d/pve-enterprise.list sed -i 's/^deb/#deb/' /etc/apt/sources.list.d/ceph.list # Add no-subscription repo echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" \ > /etc/apt/sources.list.d/pve-no-sub.list # Update apt update && apt dist-upgrade -y
3

Add 500GB NVMe as VM storage

The Windows VM NVMe (500GB) is not part of the Proxmox OS install. Add it as a raw LVM volume for VM disk use.

# Identify the 500GB NVMe device (usually /dev/nvme1n1) lsblk # Create LVM volume group on it pvcreate /dev/nvme1n1 vgcreate vg-vm /dev/nvme1n1 # In the Proxmox web UI: # Datacenter → Storage → Add → LVM → ID: vm-nvme, Volume Group: vg-vm
4

Add IronWolf drives as ZFS pool storage

In the Proxmox web UI, navigate to: Datacenter → <hostname> → Disks → ZFS → Create ZFS. RAID level: Mirror. Select both IronWolf drives.

# Alternatively via CLI — identify drive identifiers first lsblk -o NAME,SIZE,MODEL | grep -i seagate # Create the ZFS mirror pool (replace sdX with actual device letters) zpool create -f datapool mirror /dev/sda /dev/sdb # Verify pool health zpool status datapool # In Proxmox web UI, add as storage: # Datacenter → Storage → Add → ZFS → ID: datapool, Pool: datapool
Use /dev/disk/by-id/ paths rather than /dev/sda for ZFS pools — drive letters can shift across reboots. Run ls /dev/disk/by-id/ | grep -v part to find them.

ZFS pool — IronWolf Mirror

The two 8TB Seagate IronWolf drives run as a ZFS Mirror (RAID-1). 8TB usable with full redundancy — one drive can fail with zero data loss. Expandable to RAID-Z1 and RAID-Z2 by adding drives later.

1

Create ZFS Mirror pool (CLI method)

Always use /dev/disk/by-id/ paths for ZFS — stable identifiers that won't shift on reboot.

# List drives by ID ls /dev/disk/by-id/ | grep -i seagate | grep -v part # Create mirror pool — replace wwn-... with your actual IDs zpool create -f -o ashift=12 datapool mirror \ /dev/disk/by-id/wwn-0x5000c500xxxxxxxx \ /dev/disk/by-id/wwn-0x5000c500yyyyyyyy # Verify zpool status datapool zpool list datapool
ashift=12 sets 4K sector alignment for the IronWolf drives — always specify this at pool creation; it cannot be changed after.
2

Set ZFS datasets for shares

# Create datasets for each share category zfs create datapool/media zfs create datapool/projects zfs create datapool/documents zfs create datapool/backups zfs create datapool/docker # Docker volumes # Enable compression (LZ4 — fast, ~1.5x ratio for media) zfs set compression=lz4 datapool # List all datasets zfs list
3

Configure ZFS ARC cache size

With 64GB RAM, ZFS ARC can grow large. Limit it so the VMs always have their allocated memory.

# Set ARC max to 14GB (allows ZFS to use up to 14GB of free RAM for read cache) echo "options zfs zfs_arc_max=15032385536" > /etc/modprobe.d/zfs.conf update-initramfs -u
4

Enable weekly scrubs

# Proxmox includes a weekly scrub cron by default # Verify it's enabled: cat /etc/cron.d/zfsutils-linux # Run a scrub manually anytime to verify disk health: zpool scrub datapool zpool status datapool # Check after scrub completes

Intel i225-V NIC — Dedicated NAS Interface

The Intel i225-V adds a dedicated 2.5GbE port for NAS traffic. In Proxmox, create a second Linux bridge (vmbr1) on this interface and bind the Docker LXC's network interface to it for isolated, high-throughput NAS transfers — separate from VM management traffic on vmbr0.

Power budget

Corsair RM750x 750W PSU. Phase 1 runs very efficiently — power draw nearly doubles only when the GPU arrives in Phase 2.

ScenarioEst. DrawPSU Load %UPS Runtime (1500VA)
Phase 1 idle (no GPU)~80W11%—
Phase 1 peak (no GPU)~150W20%—
Phase 2 idle (with GPU)~150W20%~60 min
Phase 2 peak (GPU rendering)~450W60%~12–15 min
The APC Back-UPS Pro 1500VA is added in Phase 2 when power draw nearly doubles. Sizing it once for the full system avoids replacing a smaller UPS later.

Storage expansion

The Fractal Define 7 supports up to 14× 3.5" drives in storage layout. The MSI B650 has 4× SATA ports — drives 5+ require a PCIe SATA expansion card.

StageDrivesZFS ConfigUsableFailure Tolerance
Launch2× 8TBMirror (RAID-1)8TB1 drive
Expand3× 8TBRAID-Z116TB1 drive
Protect4× 8TBRAID-Z216TB2 drives
Grow6× 8TBRAID-Z232TB2 drives
Future6× 16TBRAID-Z264TB2 drives
ZFS does not support changing RAID level in-place (e.g. Mirror → RAID-Z1). Expansion is done by creating a new pool and migrating data, or by adding a new vdev to an existing pool. Plan your topology upfront.

Key Design Decisions
DecisionReason
AM5 over AM4Every Ryzen 7000 CPU has RDNA 2 iGPU — Proxmox console stays on iGPU permanently, GPU passes cleanly to Windows VM. DDR5 bandwidth benefits 4K editing. Platform supported through 2027+.
Ryzen 9 7900 (not 7900X)65W TDP, Wraith Prism included, same 12 cores as 7900X, ~$120 cheaper. Stock cooler handles 65W fine for 24/7 server use.
2×32GB DDR5 (not 4×16GB)Leaves 2 DIMM slots free for future 128GB upgrade. Same cost as 4×16GB but preserves expandability.
Two NVMe drives500GB dedicated to Windows VM isolates I/O from Proxmox host. Better DaVinci Resolve scratch performance.
Fractal Define 714× 3.5" bays vs. Focus 2's 4-bay ceiling. $80 more but removes the case as a storage bottleneck for the lifetime of the build.
750W PSU in Phase 1Sized for Phase 2 GPU (RTX 4070 Ti Super 285W TDP). Replacing PSU later requires full teardown — buy right once.
